# B200 OnPair vs nvCOMP — full results

> PRELIMINARY: unlocked clocks (±~5%), single-invocation ranking, NCU blocked. Run with `--gpu-validate`; every OnPair decode kernel shown is byte-exact vs CPU decode (the only non-byte-exact kernels are the ablation timing-proxies, which are excluded here). Decode = GiB/s over uncompressed bytes. Generated by `b200_results.py`.

Ratio = uncompressed / compressed. OnPair `auto` = the shipped arch-aware selector's pick; `best` = fastest byte-exact kernel measured (excludes ablation proxies). nvCOMP Zstd: HW = Blackwell hardware backend (unsupported for Zstd); the reported ratio/decode is the best CUDA-backend level.
